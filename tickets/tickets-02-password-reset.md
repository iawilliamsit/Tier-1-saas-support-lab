# Ticket 02: Password Reset

## Ticket Summary
User requested a password reset for ClearDesk CRM.

## User Impact
User could not access the application and needed access restored to continue daily work.

## Priority
Medium

## Category
Account Access

## Reported Symptoms
- User forgot current password.
- User was unable to complete login.
- User confirmed access was needed for normal work tasks.

## Questions Asked
- Can you confirm your username?
- Are you able to access your company email?
- Are you receiving the password reset email?
- Are you able to complete MFA after the reset?
- Is this affecting only ClearDesk CRM or other applications too?

## Troubleshooting Steps
1. Verified the user identity using approved information.
2. Confirmed the username and email address.
3. Sent password reset instructions.
4. Confirmed the user received the reset email.
5. Had the user create a new password.
6. Confirmed the user could complete MFA.
7. Confirmed successful login.

## Result
Resolved at Tier 1.

## Customer Communication
Advised the user that the password reset was completed and confirmed they could access the application before closing the ticket.

## Resolution Code
Password Reset Completed

## Escalation Needed?
No

## Internal Notes
If the user had not received the password reset email, the next step would be to verify spam/junk folder, email access, and account status.

## Knowledge Base Opportunity
Add password reset steps to the account access guide.
